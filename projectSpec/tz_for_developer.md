## 2. ДОРАБОТКА ПУБЛИЧНОГО САЙТА

### 2.1 Кнопки «Купить билет»

Все кнопки с текстом «Купить билет», «Купить билеты», «Забронировать» на всех страницах сайта должны переводить пользователя на страницу `#tickets`. На странице `tickets` необходимо встроить виджет онлайн-оплаты, подключённый к платёжной системе ЮKassa.

После успешной оплаты клиент автоматически получает электронный билет на указанный email.

### 2.2 Формы заявок → личный кабинет

Все формы на сайте (бронирование экскурсий, театра, квестов, лекций, мастер-классов, дней рождения, партнёрств) должны отправлять данные через API в раздел «Заявки» личного кабинета, где менеджер их обрабатывает. Telegram-уведомление менеджеру сохраняется как дублирующий канал.

**Поля передаются:** имя, телефон, email, выбранное мероприятие, дата, количество человек, комментарий, источник (ID страницы, с которой подана заявка).

## 3. ЛИЧНЫЙ КАБИНЕТ — СИСТЕМА УПРАВЛЕНИЯ МУЗЕЕМ

### 3.1 Общие требования

- Веб-приложение, доступное по адресу `admin.домен.ru` или `домен.ru/admin`
- Адаптивная вёрстка (работа на ПК, планшете, смартфоне)
- Авторизация по логину и паролю
- HTTPS обязателен
- Ежедневное автоматическое резервное копирование базы данных

### 3.2 Роли пользователей

| Роль | Доступ |
|---|---|
| Администратор | Полный доступ: все разделы, настройки, пользователи, финансы |
| Менеджер | Бронирования, клиенты, мероприятия, заявки с сайта; без финансовой отчётности и управления пользователями |
| Кассир | Только продажа билетов и просмотр расписания |

---

### 3.3 Раздел: Главная панель (Dashboard)

Первый экран после авторизации. Содержит:

**Блок «Продажи»**
- Суммарная выручка за выбранный период (переключатели: сегодня / неделя / месяц / произвольный диапазон)
- Количество проданных билетов за период
- Сравнение с предыдущим аналогичным периодом: прирост или снижение в %
- Линейный график выручки по дням

**Блок «Мероприятия сегодня»**
- Список событий текущего дня: время, название, заполненность (N занято из M мест)
- Цветовая индикация: зелёный ≥ 70%, жёлтый 30–69%, красный < 30%

**Блок «Уведомления»**
- Новые заявки с сайта, ожидающие обработки
- Неоплаченные бронирования (с указанием, сколько времени прошло)
- Отменённые бронирования за последние 24 часа
- Мероприятия с заполненностью менее 30% — рекомендация к продвижению

---

### 3.4 Раздел: Заявки с сайта

Отдельный раздел для входящих заявок с публичного сайта.

Каждая заявка отображает:
- Тип (бронирование экскурсии / театра / квеста / лекции / мастер-класса / дня рождения / партнёрства)
- Имя, телефон, email клиента
- Выбранное мероприятие, дата, количество участников
- Комментарий
- Время подачи заявки
- Статус: Новая / В обработке / Переведена в бронирование / Отклонена

**Действия менеджера:**
- Перевести заявку в бронирование (автоматически создаётся карточка в разделе «Бронирования»)
- Позвонить или написать клиенту прямо из интерфейса
- Отклонить с комментарием

**Уведомления:** при поступлении новой заявки менеджер получает push-уведомление в браузере и сообщение в Telegram (бот).

---

### 3.5 Раздел: Календарь мероприятий

Интерактивный календарь с переключением вида: месяц / неделя / день.

**Функции:**
- Просмотр всех запланированных событий
- Создание нового события нажатием на ячейку даты/времени
- Перенос события перетаскиванием (drag & drop) с автоматическим уведомлением записавшихся клиентов по email
- Просмотр количества занятых и свободных мест на каждом событии
- Назначение ответственного сотрудника на событие
- Цветовая кодировка по типу мероприятия
- Фильтр по типу и ответственному

---

### 3.6 Раздел: Бронирования

Таблица всех заказов.

**Пайплайн статусов:**
`Новая → Подтверждена → Оплачена → Завершена → Отменена`

Переходы:
- Оплата онлайн через ЮKassa → автоматически переводит в «Оплачена»
- Остальные переходы — вручную менеджером

**Карточка бронирования содержит:**
- ФИО, телефон, email клиента
- Мероприятие, дата, время начала
- Количество участников: взрослые / дети (с возрастом)
- Статус и история изменений статуса с временными метками
- Способ оплаты и сумма
- Выбранные допуслуги (торт, фотограф, сувениры и пр.)
- Комментарий клиента
- Внутренние заметки сотрудника (не видны клиенту)

**Быстрый поиск:** по ФИО, телефону, email, номеру бронирования.

**Действия:**
- Изменить дату, время, состав
- Отменить бронирование (с опциональным уведомлением клиента)
- Отправить клиенту письмо: подтверждение / напоминание за 24 часа / уведомление об изменении
- Принять оплату вручную (наличные, перевод на карту)
- Распечатать или повторно отправить электронный билет

### 3.7 Раздел: Клиенты (CRM)

База посетителей, автоматически пополняемая при каждом бронировании.

**Карточка клиента:**
- ФИО, телефон, email
- Дата рождения (если указана при бронировании)
- Источник первого контакта (сайт / звонок / соцсети / рекомендация)
- История посещений: дата, название мероприятия, состав группы
- История покупок: сумма каждой покупки, способ оплаты
- Суммарный LTV (общая выручка от клиента)
- Использованные промокоды и сертификаты
- Теги (например: «VIP», «школьная группа», «постоянный клиент», «именинник»)
- Поле для заметок менеджера

**Функции:**
- Поиск по любому полю
- Фильтрация: по дате последнего визита, количеству посещений, сумме покупок, тегу
- Экспорт базы в Excel / CSV
- Массовая email-рассылка по отфильтрованному сегменту

### 3.8 Раздел: Мероприятия (Каталог программ)

Справочник всех программ музея. На основе программ создаются конкретные события в календаре.

**Карточка программы:**
- Название и описание (краткое — для карточек сайта, полное — для письма клиенту)
- Тип: экскурсия / квест / театр / лекция / мастер-класс / день рождения / партнёрское
- Фотографии (загрузка нескольких изображений)
- Возрастное ограничение
- Продолжительность в минутах
- Максимальное количество участников в группе
- Стоимость: взрослый билет / детский / льготный / групповой (цена за группу)
- Расписание по умолчанию: дни недели и время проведения
- Статус: активна / скрыта / в архиве
- Заполняемость: автоматически отображает, сколько мест продано и сколько осталось на ближайших событиях

**Допродажи (Upsell) — настройка менеджером:**

Для каждой программы менеджер может добавить перечень дополнительных услуг, предлагаемых клиенту при оформлении заказа:

| Допуслуга | Описание | Стоимость |
|---|---|---|
| 🎂 Торт | Именной торт на праздник | настраивается |
| 📸 Фотограф | Съёмка 30 или 60 минут | настраивается |
| 🎁 Набор сувениров | Брендированные сувениры | настраивается |
| 🎨 Мастер-класс | Дополнительный мастер-класс | настраивается |
| 🎟 Сертификат | Подарочный сертификат музея | настраивается |
| 👑 VIP-сопровождение | Персональный гид | настраивается |

Менеджер сам создаёт, редактирует и удаляет допуслуги, задаёт название, описание, фото и стоимость. При оформлении бронирования клиент видит доступные допуслуги и добавляет их к заказу одним кликом.

## 5. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 5.1 Стек технологий

Выбор стека остаётся за исполнителем. Рекомендуемые варианты:

**Бэкенд:** Python (Django / FastAPI) или Node.js  
**База данных:** PostgreSQL  
**Фронтенд ЛК:** React или Vue.js (или любой современный фреймворк)  
**Хостинг:** VPS (минимум 2 CPU, 4 ГБ RAM, 50 ГБ SSD)  
**Очередь задач:** Celery / Redis (для отправки писем и уведомлений)

### 5.2 Безопасность

- Все передачи данных только по HTTPS
- Пароли хранятся в хэшированном виде (bcrypt)
- Двухфакторная аутентификация для роли «Администратор»
- Защита API от CSRF и SQL-инъекций
- Ограничение числа попыток входа (rate limiting)

### 5.3 Производительность и надёжность

- Время отклика страниц личного кабинета < 1 сек
- Ежедневное автоматическое резервное копирование БД
- Мониторинг доступности (uptime) с уведомлением при падении
- Логирование всех действий пользователей ЛК

### 5.4 Браузеры и устройства

Публичный сайт и личный кабинет должны корректно работать в: Chrome, Firefox, Safari, Edge (последние 2 версии). Адаптивная вёрстка для устройств с шириной от 320px.
